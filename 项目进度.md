### 9. MCTS 算法升级 (Phase 8) — 2026-04-26

#### 9a. IMCTS 调研
- 调研 Implicit Minimax MCTS (Lanctot 2014)、Silverfish (C# 最强规则AI)、ByteRL (字节端到端策略网络)
- 三种不同 "IMCTS" 算法对比，确定采用 Lanctot 2014 方案

#### 9b. P0: Implicit Minimax Backup — `deb1361`
- `node.py`: 新增 `h_value` 字段 (minimax heuristic)
- `backprop.py`: Q用平均 + h用minimax 双路径备份
- `uct.py`: selection 时混合 Q+h，β = β_initial/(1+visits)
- `config.py`: `use_imcts=True`, `imcts_beta=0.5`

#### 9c. P2: Obliged Actions — `7a33ff1`
- `expansion.py`: 检查 pruner.get_obliged_actions()，自动选择强制/唯一动作
- 减少 MCTS 预算浪费在无意义分支上

#### 9d. P3: 2D Hash Transposition — `1207de2`
- `transposition.py`: 双哈希 — IS hash (信息集) + full hash (含手牌)，碰撞检测
- 修复手牌不在 hash 中的 bug，新增 `compute_full_hash()`

#### 9e. P1-lite: Chance Node Dedup — `b8aa317`
- `expansion.py`: 随机结果按 state_hash 去重，复用已有 outcome

#### 9f. P4: Bayesian 对手威胁集成 — `064cab7`
- `opponent_simulator.py` (180→330行): 接入 `BayesianOpponentModel`
  - 扫描预测手牌中的伤害法术/解场/AOE/武器
  - `classify_card_threat()`: 基于卡牌文本分类威胁类型
  - 按手牌比例缩放，无模型时回退到启发式
- `submodel.py`: `eval_threat()` 增加 `_bayesian_spell_threat()` 贝叶斯法术威胁
- `engine.py`: evaluate_leaf 前将 `_bayesian_model` 注入 leaf_state
- SimulatedOpponentTurn 新增 `removal_threat`, `predicted_damage_cards`, `predicted_removal_cards`

**Phase 8 测试基线: 756 pass, 0 regressions**

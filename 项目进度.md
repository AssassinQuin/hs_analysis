## 🏗️ 架构决策摘要

| # | 决策 | 理由 |
|---|------|------|
| D017 | IMCTS h_value 与 Q 双路径 | Lanctot 2014 验证，增强决策质量 |
| D018 | β/(1+visits) 衰减混合 | 早期信任启发，后期信任蒙特卡洛 |
| D019 | IS hash + full hash 双哈希 | DUCT 跨世界共享 + 碰撞检测 |
| D020 | P1 简化为 outcome dedup | 完整 CNB 架构不支持 |
| D021 | 贝叶斯威胁通过 state 属性传递 | 零侵入集成 |
| **D022** | **统一引擎 engine/ 顶层包** | **替代散落 search/ 的 mechanic 模块** |
| **D023** | **abilities/ 提升为顶层包** | **声明式能力系统，为 JSON + I-MCTS 准备** |
| **D024** | **DeterministicRNG 替代 random** | **MCTS 确定性要求，xorshift32 可复现** |
| **D025** | **分发表替代 if-chain** | **36 EffectKind 注册表，可扩展** |
| **D026** | **零 copy-eval 目标选择** | **MCTS 性能瓶颈，每展开 24-40 次深拷贝→0** |

完整决策记录: `docs/architecture/DECISIONS.md`

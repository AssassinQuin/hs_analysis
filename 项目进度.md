## ✅ 已完成任务

### 1. V11 RHEA 引擎核心
- **cross-turn planning** — 跨回合规划，决策考虑后续回合影响
- **adaptive time budget** — 自适应时间预算 (3-15s)
- **多对局检测** — 仅 `GameState.DebugPrintPower()` 的 `CREATE_GAME` 触发重置，避免双重复位
- **对手手牌情报4层架构**:
  - 确认手牌 (SHOW_ENTITY → HAND, X%)
  - 回手卡牌 (PLAY → HAND, 100%)
  - 入墓卡牌 (PLAY → GRAVEYARD)
  - 贝叶斯预测 (>50% 概率)
- **Config 集中化重写** — `analysis/config.py` 统一管理引擎参数
- **死代码清理** — 移除 `_player_entities`, `_in_full_entity`, `_in_show_entity`, `controller_map`, `_update_controller()` 等
- **诊断零错误** — 全部文件通过 IDE 诊断检查

### 2. 名称归一化
- **新增 `analysis/utils/player_name.py`**:
  - `normalize_player_name()` — 清洗空字符串/UNKNOWN/零宽字符/BOM
  - `is_anonymous_name()` — 检测匿名玩家 (UNKNOWN 开头)
  - `name_matches()` — BattleTag 匹配 + 大小写不敏感 + 后缀剥离
    - `"Player#1234"` 匹配 `"Player"`
    - `"Player#1234"` **不**匹配 `"Player#5678"`
- **GameReplayer 集成** — `_handle_game_debug_line()` 从 `DebugPrintGame()` 的 `PlayerName=` 自动检测玩家名
- **PacketReplayer 集成** — `_handle_create_game()` 使用 `normalize_player_name` + `is_anonymous_name` 替代 `startswith("UNKNOWN")`
- **replay_game.py** — `--player` 默认值改为空字符串（自动检测），移除硬编码 `"湫然#51704"`

### 3. Card 模型方法迁移
- **Card 新增方法** (`analysis/models/card.py`):
  - `has_mechanic(keyword)` — 快速 mechanics 检查
  - `get_effects()` — 委托给 `card_effects.get_effects()`
  - `compute_mechanics()` — 集成 `card_cleaner.extract_mechanics()`
  - `effective_overload()` / `effective_armor()` — 结构化字段 + 文本正则回退
  - `total_damage()` — direct + random damage 快捷访问
  - `is_minion` / `is_spell` / `is_weapon` / `is_hero` / `is_location` — 类型谓词
  - `mechanics_set` — 缓存的 set 属性
- **Minion 新增属性** (`analysis/search/game_state.py`):
  - `is_friendly` / `is_enemy` — 归属判断
  - `can_attack_now` — 综合攻击可用性 (冻结/风怒/休眠)
  - `total_stats` — attack + health 总和
- **调用方迁移** — `rhea_engine.py`, `strategic.py`, `hero_card_handler.py` 已改用 Card 实例方法

### 4. 新增模块
- `analysis/data/card_effects.py` — 结构化效果提取 (damage/aoe/heal/armor/draw/summon/buff/discard/cost_reduce)
- `analysis/constants/hs_enums.py` — 枚举定义
- `analysis/constants/__init__.py`

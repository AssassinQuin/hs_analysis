### 1. peter1591/hearthstone-ai ⭐⭐⭐⭐⭐

| 项目 | 详情 |
|------|------|
| URL | https://github.com/peter1591/hearthstone-ai |
| Stars | 326 |
| 语言 | C++14/17 (引擎) + TensorFlow (神经网络) + C# (UI/日志) |
| 最近更新 | 2018-03（已归档） |
| 许可 | GPL |

**简介**：AlphaGo 风格的炉石 AI 完整管线。Monte Carlo Tree Search + 深度神经网络 + 自定义高性能游戏引擎。由 Google 工程师开发，专门研究含隐藏信息的 MCTS。

**核心算法**：
- **MO-MCTS (多观察者 MCTS)**：两位玩家各维护独立的游戏树，处理隐藏信息（对手手牌、牌库）
- **神经网络状态价值函数**：TensorFlow 模型从局面估算胜率（~79% 准确率）
- **早停模拟**：用神经网络提前终止模拟 rollout，大幅提升效率
- **重定向节点**：策略融合技术，减少搜索树规模
- **AlphaZero 风格自我对弈训练管线**

**与 6 项需求的相关性**：

| 需求 | 评分 | 说明 |
|------|------|------|
| 抉择分析 | ⭐⭐⭐ | MCTS 框架天然评估和排序所有可能操作 |
| 斩杀检测 | ⭐⭐⭐ | DFS + 状态价值函数 + MCTS 模拟自动发现斩杀线 |
| 反杀概率 | ⭐⭐⭐ | 神经网络直接输出胜率估算 |
| 对手手牌 | ⭐⭐⭐ | MO-MCTS 显式建模隐藏信息 |
| 交互分析 | ⭐⭐ | 游戏引擎模拟所有交互但未显式分类 |
| 卡牌估值 | ⭐⭐⭐ | 状态价值网络隐式评估卡牌贡献 |

**借鉴价值**：🥇 **最高** — MO-MCTS 处理隐藏信息的架构是本项目对手预测的理论基础，状态价值函数是局面评估的参考实现。

## 战略建议

### 按需求优先级推荐

| 需求 | 最佳参考项目 | 推荐理由 |
|------|-------------|---------|
| **抉择分析** | hearthstone-ai + SilverFish | MO-MCTS 架构 + 启发式评分实战 |
| **斩杀检测** | HS_SPR_CAL + hearthstone-ai | 穷举搜索 + MCTS 组合 |
| **反杀概率** | hearthstone-ai | 神经网络胜率估算（79%准确率） |
| **对手手牌** | hearthstone-ai (MO-MCTS) | 隐藏信息建模的唯一完整参考 |
| **交互分析** | fireplace + RosettaStone | 最完整的卡牌交互模拟 |
| **卡牌估值** | card2code + hearthstone-ai | 形式化卡牌表示 + 隐式价值网络 |

### 按技术路线推荐

**如果走 Python 路线（方案 B，推荐）：**

1. **fireplace** — 游戏模拟引擎（Python 原生）
2. **python-hearthstone** — 卡牌数据 + 日志解析
3. **hsdata** — 自动更新的原始数据
4. **card2code** — 卡牌文本形式化参考
5. **hearthstone-ai** — MO-MCTS + 神经网络架构参考

**如果走 C#/HDT Fork 路线（方案 A）：**

1. **SabberStone** — C# 游戏引擎 + 评分制 AI
2. **SilverFish** — 启发式卡牌评分实战参考
3. **Hearthstone-Script** — MCTS 实时决策参考
4. **hearthrock** — Web API 架构参考

### 实施优先级建议

1. 📖 **先研究** hearthstone-ai 的 REPORT.md — 理解 MO-MCTS 理论
2. 📖 **先研究** HS_SPR_CAL 的穷举搜索 — 理解斩杀检测方法论
3. 🔧 **后搭建** 基于 fireplace 的 Python 模拟环境
4. 🧠 **再实现** MO-MCTS 决策框架
5. 📊 **再实现** 卡牌价值模型（V2 模型集成）
6. 🔌 **最后** 游戏接入（HDT 插件 或 日志解析）

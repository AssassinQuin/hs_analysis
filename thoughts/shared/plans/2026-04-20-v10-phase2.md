## Batch 2: BattlecryDispatcher 战吼派发

### Task 2.1: `hs_analysis/search/battlecry_dispatcher.py` (新建, ~250 行)
- 复用 `spell_simulator.py` 的 `SPELL_EFFECT_PATTERNS` 和 `EffectParser`
- `dispatch_battlecry(state, card, minion) → GameState`
  - 解析 card.text 中的战吼效果
  - 按效果类型分发：伤害/治疗/召唤/抽牌/buff/发现
  - 目标选择：greedy（枚举合法目标，选 evaluate 最高的）
- 支持的战吼模式：
  - `战吼：造成N点伤害` → 敌方最优目标
  - `战吼：恢复N点生命` → 己方最优目标
  - `战吼：召唤N/N` → board 最优位置
  - `战吼：抽N张牌` → apply_draw
  - `战吼：+N/+N` → buff self 或 target
  - `战吼：发现...` → 委托 discover framework（Phase 5 前，先返回原始 state）
  - `战吼：消灭...` → destroy target
  - `战吼：冻结...` → freeze target

### Task 2.2: 修改 `rhea_engine.py` — PLAY MINION 接入点
- 在 `apply_action` 的 PLAY MINION 分支，minion 插入 board 后：
  ```python
  # 新增：战吼派发
  try:
      from hs_analysis.search.battlecry_dispatcher import dispatch_battlecry
      s = dispatch_battlecry(s, card, new_minion)
  except Exception:
      pass
  # 新增：触发器
  try:
      from hs_analysis.search.trigger_system import TriggerDispatcher
      s = TriggerDispatcher().on_minion_played(s, new_minion, card)
  except Exception:
      pass
  ```

### Task 2.3: `hs_analysis/search/test_battlecry_dispatcher.py` (新建, ~150 行)
- 战吼伤害：play card → 敌方 minion 扣血
- 战吼治疗：play card → 己方 hero/minion 回血
- 战吼召唤：play card → board 多一个 token
- 战吼抽牌：play card → hand 多牌
- 战吼 buff：play card → minion 攻击/血量增加
- 无战吼卡：play card → 无额外效果（回归安全）
- ~8 tests

### 验收: 284+ 原有 + ~8 新 = 292+ tests passing

## Batch 3: DeathrattleQueue 亡语队列

### Task 3.1: `hs_analysis/search/deathrattle.py` (新建, ~200 行)
- `resolve_deaths(state, max_cascade=5) → GameState`
  - 收集双方 board 上 health <= 0 的 minion
  - 按板位顺序（左到右）排亡语队列
  - 执行每个亡语效果（复用 battlecry_dispatcher 的效果解析模式）
  - 重新检查死亡（级联），最多 max_cascade 层
  - 从 board 移除死亡 minion
- 亡语效果模式：
  - `亡语：对随机敌人造成N点伤害`
  - `亡语：召唤N/N`
  - `亡语：抽N张牌`
  - `亡语：+N/+N` (buff 友方)

### Task 3.2: 修改 `rhea_engine.py` — ATTACK 接入点
- 替换 inline `s.board = [m for m in s.board if m.health > 0]`
- 改为 `from deathrattle import resolve_deaths; s = resolve_deaths(s)`
- 同时处理敌方和友方死亡

### Task 3.3: `hs_analysis/search/test_deathrattle.py` (新建, ~150 行)
- 亡语伤害：minion 死亡 → 敌方受伤
- 亡语召唤：minion 死亡 → 召唤 token 到原位
- 亡语抽牌：minion 死亡 → 手牌增加
- 亡语级联：A 亡语杀 B → B 亡语触发 → 正确执行
- 级联上限：5 层后停止
- 无亡语 minion：死亡 → 正常移除，无额外效果
- ~8 tests

## Batch 4: AuraEngine 光环引擎

### Task 4.1: `hs_analysis/search/aura_engine.py` (新建, ~200 行)
- `recompute_auras(state, max_iterations=10) → GameState`
  - 移除所有 `trigger_type="aura"` 的 enchantment
  - 遍历 board 上有光环关键词的 minion
  - 根据光环类型生成 enchantment 并 apply 到目标
  - 重复直到稳定或达到 max_iterations
- 光环目标类型：`all_friendly`, `adjacent`, `self`, `hand_cards`
- 光环效果解析（正则）：
  - `你的其他随从获得+N攻击力` → attack_delta to all_friendly
  - `相邻随从获得+N/+N` → attack+health to adjacent
  - `你的手牌费用减少N` → cost_delta to hand_cards

### Task 4.2: 修改 `rhea_engine.py` — 全局接入
- 在 PLAY MINION、ATTACK（死亡后）、PLAY SPELL 后调用 `recompute_auras(s)`
- 用 try/except 包裹，graceful degradation

### Task 4.3: `hs_analysis/search/test_aura_engine.py` (新建, ~120 行)
- 攻击力光环：minion 入场 → 友方全部 +attack
- 相邻光环：minion → 左右邻居 buff
- 死亡后重算：光环 minion 死亡 → buff 消失
- 无限循环保护：max_iterations 生效
- ~6 tests

### 验收: 300+ 原有 + ~6 新 = 306+ tests passing

## Batch 5: DiscoverFramework 发现框架

### Task 5.1: `hs_analysis/search/discover.py` (新建, ~300 行)
- `resolve_discover(state, card_text, hero_class) → GameState`
  - 解析发现池约束：`发现一张法术` → spells, `发现一张野兽` → beasts
  - 从 unified_standard.json 生成候选池（同职业 + 中立）
  - 随机抽 3 张候选
  - 对每张用 `quick_eval(state_with_card)` 打分
  - 选择最高分，加入手牌
  - 池为空时 fallback：加 1/1 白板
- `generate_discover_pool(hero_class, card_type=None, race=None) → List[Card]`
  - 懒加载 unified_standard.json，缓存 pool
  - filter by class + neutral + constraints

### Task 5.2: 战吼中发现 → 委托 discover
- 修改 `battlecry_dispatcher.py` 中发现类型战吼
- 调用 `resolve_discover(state, text, hero_class)`

### Task 5.3: `hs_analysis/search/test_discover.py` (新建, ~120 行)

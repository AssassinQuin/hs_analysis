# 外部研究资料: MCTS/UCT 在炉石传说中的应用

> 生成日期: 2026-04-24
> 检索范围: MCTS/UCT 算法在炉石传说等卡牌游戏中的实现方案

## 来源1: Enhancing Monte Carlo Tree Search for Playing Hearthstone

- **作者**: Jean Seong Bjorn Choe, Jong-Kook Kim (Korea University)
- **会议**: IEEE Conference on Games (COG) 2020
- **URL**: https://ieee-cog.org/2020/papers2019/paper_257.pdf
- **检索日期**: 2026-04-24
- **可信度**: 高 (IEEE 会议论文，经过同行评审)

### 关键发现

#### 1.1 DAG 搜索空间 + 状态抽象
- 使用状态抽象 (State Abstraction) 将搜索空间表示为有向无环图 (DAG) 而非树
- 通过转置表 (Transposition Table) 实现，键为抽象后的状态
- 将可观察信息与隐藏信息分离，每个节点只包含活跃玩家的信息集
- 效果：大量相似状态被聚合，有效减少搜索空间

#### 1.2 修改版 UCD 算法
- 基于 Saffidine 等人提出的 UCD (Upper Confidence bound for DAG)
- 将统计信息存储在**边**而非节点上
- 选择公式: `argmax_{e∈C(s)} Q_{d1}(s(e)) + c * sqrt(2*ln(N(p(e))) / n_{d2}(e))`
  - d1, d2: 可调深度参数
  - Q: 累积奖励值, n: 访问计数, N: 兄弟边总访问计数
- 反向传播: 递归更新路径上和非路径上的节点/边统计

#### 1.3 稀疏采样 + 阻尼采样 (Damped Sampling)
- 机遇节点 (Chance Node) 使用稀疏采样 (Sparse Sampling) 控制分支数
- 阻尼采样 (DS): 阈值随已遇到的机遇节点数递减
  - 公式: `k = floor(C * n^α * η^{-μ})`
  - C, α: DPW 参数; η: 阻尼参数; μ: 已遇到的大分支机遇节点计数
- 效果：深度搜索时有效减少随机分支因子

#### 1.4 指数递减时间分配
- 每个原子动作的搜索时间递减: `T(n) = T0 * γ^{n-1}`
- γ 通过约束 `Tmax = Σ_{i=1}^{k} T0 * γ^{i-1}` 求解
- 参数: k=3-5, Tmax=10-30s, Tmin=1.5-3s, T0=5-10s

#### 1.5 领域知识启发
- **基于类别的动作过滤**: 根据卡牌文本信息分类，过滤自毁/浪费性动作
- **必要动作 (Obliged Actions)**: 无条件有利动作（如无障碍攻击对手英雄），移除 END_TURN 选项强制执行

#### 1.6 实验结果
- 打败 TycheAgent (CIG'18 第三名): MidRangeJade 66%, AggroPirate 78%
- 最佳 UCD 参数: d1=2, d2=1 (Hunter), d1=2, d2=2 (Paladin)
- 最佳 DPW 参数: C=0.15-0.6, α=0.5
- UCT 常数: c=0.25 (Hunter), c=0.4 (Warlock), c=0.8 (Paladin)

## 来源2: A Practical Solution to Handling Randomness and Imperfect Information in MCTS

- **作者**: Maciej Świechowski (QED Software), Tomasz Tajmajer (University of Warsaw)
- **会议**: FedCSIS 2021 (Annals of Computer Science and Information Systems, Vol. 25)
- **URL**: https://annals-csis.org/Volume_25/drp/pdf/3.pdf
- **检索日期**: 2026-04-24
- **可信度**: 高 (经过同行评审，实际 HS AI 竞赛参与者)

### 关键发现

#### 2.1 双接口设计
- **GS-state (Game Simulation State)**: 完整游戏逻辑接口，用于模拟
- **IS-state (Information Set State)**: 玩家可见信息，用于决策和统计
- 关键: IS-state 可以省略不重要的可见信息 (如用桶代替精确数值)，减少状态空间
- IS-state 是静态快照，不实现动作执行

#### 2.2 完全消除自然行动
- 不创建单独的机遇节点 (Chance Node)
- 随机性在每次迭代开始时通过确定化 (Determinization) 处理
- 搜索树只包含决策节点
- 优势：简化树结构，避免随机事件的组合爆炸

#### 2.3 多层转置表
- 两层结构:
  1. 第一层: 基于 IS-state 的哈希查找 → MCTS 节点
  2. 第二层: 同一 IS-state 对应多个不同 GS-state 的统计
- 效果：同信息集的状态共享统计，不同确定化的结果也被考虑

#### 2.4 MCTS 迭代流程 (伪代码)
```
procedure ITERATE(gs_state):
  rootNode ← updateRoot(gs_state)
  while time available:
    gs_movingState ← determinize(gs_state)
    while selection RUNNING:
      moves ← gs_movingState.getMoves()
      for each move:
        edge ← findOrCreateEdge(move)
        edge.N ← +1
      chosenEdge ← selection(edges) // UCT
      chosenEdge.V ← +1
      if chosenEdge.V == 1: FINISHED
      gs_movingState.apply(chosenMove)
      is_state ← createInformationSet(gs_movingState)
      nextNode ← tt.findOrCreate(is_state)
    propagate(simulate(gs_movingState))
```

#### 2.5 信息集构造 (炉石特化)
- 全局公开信息: 活跃玩家、游戏阶段
- 每个玩家数据: 多态结构 (ISAnyPlayer / ISObservedPlayer)
  - ISAnyPlayer: HP、护甲、法力、场上随从 (按 ID 排序忽略位置)
  - ISObservedPlayer: 额外包含手牌和奥秘 (仅当该玩家是活跃方时)
- 优化: 随从按 ID 排序比较，位置仅在影响属性时考虑

## 来源3: Improving CCG AI with Heuristic Search and Machine Learning Techniques

- **作者**: Shuyi Zhang (MSc thesis, University of Alberta, supervised by Michael Buro)
- **年份**: 2017
- **URL**: https://ualberta.scholaris.ca/bitstreams/64998509-74b7-4bd0-8c6a-995807642c00/download
- **检索日期**: 2026-04-24
- **可信度**: 高 (Alberta 大学论文，Michael Buro 指导，CIG 2017 发表)

### 关键发现

#### 3.1 确定化 UCT (DUCT)
- 预采样若干"世界" (worlds) 从当前信息集
- 每次迭代选择一个世界，沿着兼容的子树遍历
- 多个世界共享相同动作时，聚合统计用于 UCB1 选择
- 最终选择: 根节点访问次数最多的动作

#### 3.2 机会节点分桶 (Chance Node Bucketing, CNB)
- 将相似的机会事件分组到桶中
- 每个桶预采样少量事件代表整个桶
- 分桶标准: 按卡牌类型和法力消耗
  - 例: 3个桶 × 2个预采样 = 6个后继 vs 原始12个 (50%减少)
- 有效减少抽卡、随机召唤等高分支因子节点

#### 3.3 时间预算策略
- **Multiple-Search**: 每个动作分配递减时间
  - `Time(ni) = min(max(T * α_m, LB(ni)), T * β)`
  - LB(n) = τ * T / BF(n) (BF = 分支因子)
- **One-Search**: 单次搜索，递归选择最多访问子节点
  - 若访问计数 < ψ 的节点出现，从其前驱开始 Multiple-Search
- 两种策略都给早期动作分配更多时间

#### 3.4 高级策略网络 (Rollout Improvement)
- CNN+Merge 架构: 盘面模块 (卷积) + 手牌模块 (卷积) → 合并 → 输出
- DNN+Merge 架构: 展平1D向量 → 全连接 → 合并
- 训练数据: 人类玩家对局
- 功能: 策略网络 (选择有前景动作) + 默认网络 (模拟阶段) + 结果预测 (早期截断)

#### 3.5 动作等价性
- **严格等价 (Strict Equality)**: 所有属性递归比较
- **软等价 (Soft/Hash Equality)**: 仅哈希部分属性 (忽略不影响决策的属性)
- 用于不同世界间聚合统计

#### 3.6 实验结果
- DUCT vs Silverfish:
  - Mech Mage: 57% (d=3, c=1, numWorlds=5)
  - Hand Lock: 64% (d=3, c=1, numWorlds=7)
  - Face Hunter: 68% (d=3, c=1, numWorlds=11)
- CNB 提升: 稳定 2-5% 胜率提升
- 高级策略网络: 预测准确率 35-45% (top-1), 显著高于随机

## 来源4: peter1591/hearthstone-ai (GitHub 开源项目)

- **作者**: peter1591
- **URL**: https://github.com/peter1591/hearthstone-ai
- **语言**: C++
- **检索日期**: 2026-04-24
- **可信度**: 中 (开源项目，AlphaGo 架构的炉石实现)

### 关键发现

#### 4.1 多观察者 MCTS (MOMCTS)
- 处理不完全信息: 每个玩家有独立观察
- 共享相同盘面的树节点
- 不同玩家视角的信息集分别维护

#### 4.2 大分支因子处理
- 抽卡时分枝因子可达 ~4000
- 动态概率: 基于剩余思考时间和当前展开进度决定是否继续选择
- 不要求所有子节点展开后再用 UCB

#### 4.3 AMAF/RAVE
- All-Moves-As-First / Rapid Action Value Estimation
- 跨节点信息共享: 即使不同状态的节点也能利用相似动作的统计
- 加速收敛，特别是在访问次数少的节点

#### 4.4 虚拟损失 (Virtual Loss)
- 多线程并行搜索支持
- 正在探索的节点增加虚拟损失，避免其他线程重复探索
- Parallel MCTS: 根并行 + 叶并行

#### 4.5 神经网络集成
- 策略网络: 选择有前景的动作 (更高概率)
- 默认网络: 模拟阶段的走子策略
- 结果预测: 早期截断，减少无效模拟

## 综合发现

### 跨源验证的关键结论

1. **DAG + 状态抽象 > 纯树搜索** (来源1, 2 验证)
   - 转置表减少重复状态
   - 信息集抽象降低搜索空间维度

2. **DUCT 是处理炉石不完全信息的实用方案** (来源3, 4 验证)
   - numWorlds=5-11 是合理范围
   - 统计聚合 (软等价) 提升效率

3. **渐进展开/稀疏采样是必需的** (来源1, 3 验证)
   - 炉石单回合动作空间可达数千
   - CNB (来源3) 和 DPW+DS (来源1) 是两种有效方案

4. **Rollout 策略不需要完整游戏模拟** (来源3, 4 验证)
   - 可用启发式评估函数替代
   - 随机 rollout + 动作过滤已足够有效

5. **时间分配应给早期动作更多预算** (来源1, 3 验证)
   - 指数递减 (来源1) 和分支因子自适应 (来源3)
   - 两种策略都证明有效

6. **领域知识对 MCTS 性能至关重要** (来源1, 3 验证)
   - 动作过滤减少 30-50% 无效探索
   - 评估函数质量直接决定 rollout 质量

## 经验与约束

### 架构决策 (来自 DECISIONS.md)
- D009: 三阶段渐进式改进，不是完全重写
- D017: Effect string protocol 用于跨模块分发
- D019: 所有分发点用try/except实现优雅降级
- D022: 任务进度通过action type匹配（非事件系统）

### 已知限制
- 新增追踪关注点 = 4+文件变更（GlobalGameState→OpponentState→StateBridge→GameState.copy()）
- GlobalGameState 33字段中有7+个player/opponent对偶字段，无双边抽象
- MechanicsState与GameState原始字段双重存储需同步
- StateBridge用hasattr()防护新增字段，表示schema未强制执行
- GlobalTracker直接访问BayesianOpponentModel私有方法_deck_name()

### 设计约束
- Python 3.10+，无外部依赖（纯标准库 + 项目内部模块）
- 追踪层在实时对局中运行，性能敏感
- 搜索引擎 GameState.copy() 每秒被调用数千次（MCTS树展开）
- 向后兼容：packet_replayer路径已完整，不应破坏

## 1. 游戏区域与卡牌类型

### 1.1 游戏区域

炉石传说有6个游戏区域，卡牌在区域间按固定方向流动：

| 区域 | 容量上限 | 说明 |
|------|----------|------|
| **牌库（Deck）** | 无上限（标准30张） | 从顶部抽牌，洗牌效果随机重排 |
| **手牌（Hand）** | **10张** | 超出部分直接进入墓地（不触发弃牌效果） |
| **战场（Play/Board）** | **7个位置**（随从+地标共享） | 随从和地标占据位置，武器/奥秘不占 |
| **奥秘区（Secret Zone）** | **5个不同奥秘** | 战场子区域，奥秘激活后消耗 |
| **墓地（Graveyard）** | 无上限 | 已死亡/使用/烧毁的卡牌，不参与游戏 |
| **死亡事件缓存** | 独立于墓地 | 复活效果检查此缓存而非墓地 |

**区域流动规则：**
- 卡牌移动方向：牌库 → 手牌 → 战场 → 墓地
- 跨区域移动时附魔脱落（手牌buff在进入战场后保留）
- 墓地中的卡牌视为"已死亡"，可被复活效果选中
- 被烧毁（手牌满时抽到的牌）进入墓地但**不触发亡语**
- 全区域溢出规则：目标区域满时，卡牌直接死亡（Full Zone Instant Removal）

### 1.2 随从牌（Minion）

**基本属性：** 攻击力(Attack)、生命值(Health)、法力消耗(Cost)

**召唤规则：**
- 从手牌打出时消耗法力，放置在玩家选择的战场位置（0-当前随从数）
- 场上最多 **7个随从+地标**（共享上限）
- 新随从可插入任意位置（左端、右端、或中间）
- 召唤失调（Summoning Sickness）：新召唤的随从**当回合不能攻击**，除非具有冲锋或突袭

**随从种族（Type/Tribe）：**
- 当前标准模式种族：野兽、恶魔、龙、元素、鱼人、机械、娜迦、海盗、图腾、无
- 部分随从具有**双种族**（如巨型首领既是龙又是元素）
- 种族协同影响延系(Kindred)和部分战吼/亡语效果

**放置位置的重要性：**
- 嘲讽随从的位置不影响其功能
- 光环效果（如团队领袖）对**相邻随从**生效时位置关键
- 亡语召唤的随从通常出现在最右端
- 巨型(Colossal)的附肢出现在主体的右侧

### 1.3 法术牌（Spell）

**法术派系（Spell School）：**
| 派系 | 代表色 | 示例职业 |
|------|--------|----------|
| 奥术(Arcane) | 紫色 | 法师 |
| 火焰(Fire) | 红色 | 法师、术士 |
| 冰霜(Frost) | 蓝色 | 法师、死亡骑士 |
| 神圣(Holy) | 金色 | 牧师、圣骑士 |
| 自然(Nature) | 绿色 | 德鲁伊、萨满 |
| 暗影(Shadow) | 紫色 | 牧师、术士 |
| 邪能(Fel) | 绿色 | 恶魔猎手 |
| 无(General) | 灰色 | 所有职业 |

**目标规则：**
- 定向法术需要玩家选择目标
- AoE法术自动作用于所有合法目标
- 随机目标法术由系统随机选择
- **法术伤害修正（Spell Damage）：** 场上每个"法术伤害+N"随从使你的伤害法术+N点伤害，可叠加
- 法术伤害影响总伤害量（如奥术飞弹的总伤害池增加，再随机分配）

**法术与触发器：**
- "每当你施放法术"触发器在法术**开始施放时**触发（如法术反制）
- "在你施放法术后"触发器在法术**完全结算后**触发（如狂野火法师）

### 1.4 武器牌（Weapon）

**装备规则：**
- 装备新武器时，旧武器被销毁（触发旧武器的亡语）
- 武器为英雄提供攻击力，**仅在己方回合生效**
- 英雄用武器攻击时，造成武器攻击力等量的伤害

**耐久度（Durability）：**
- 每次英雄攻击消耗1点耐久度
- 耐久度降为0时武器自动销毁
- 被攻击（对方打脸）**不消耗**耐久度
- 部分效果可增加或恢复耐久度

**关键交互：**
- 英雄无武器时攻击力为0（除非有临时攻击力buff）
- 武器攻击力在对方回合"收起"（变回0），但若被误导弹回己方随从，武器攻击力仍会造成反击伤害

### 1.5 英雄牌（Hero Card）

**替换规则：**
- 替换当前英雄，保留当前生命值和最大生命值
- 获得牌面护甲值（直接叠加到现有护甲上）
- 替换英雄技能为新技能（不再可用原技能）
- 英雄牌具有**战吼**效果

**特殊英雄牌：**
- 死亡之翼·毁灭者（CATACLYSM）：10费，12护甲，2费技能（+5攻击力），兆示升级灾变效果
- 加拉克苏斯（术士）：替换英雄，3/2武器，2费技能（召唤6/6地狱火）

### 1.6 地标牌（Location）

**基础规则（Castle Nathria引入）：**
- 放置在战场上，与随从**共享7个位置**
- 有耐久度（使用次数），无攻击力/生命值
- **无法被普通手段伤害或攻击**
- 激活技能**免费**（0法力），但消耗1耐久度，且有**1回合冷却**
- 可在打出当回合立即使用（无召唤失调）

**目标交互：**
- 地标可以指定"魔免"（无法被法术/英雄技能指定）的随从为目标
- 多个同名地标可同时存在
- 耐久度归零时地标销毁
- Patch 32.0+：地标最后一费用于召唤随从时，地标先移除腾出位置再召唤

### 1.7 起手与硬币

**起手规则：**
- 先手玩家保留3张手牌，后手玩家保留4张
- 起手阶段可选择保留或替换任意数量的卡牌
- 被替换的卡牌洗回牌库，随机抽取替换牌
- 任务牌始终出现在起手手牌中

**硬币（The Coin）：**
- 后手玩家获得1张0费法术"幸运币"（获得1个临时法力水晶）
- 硬币是法术牌，触发所有法术相关触发器
- 硬币不增加最大法力值，只临时增加可用法力

## 2. 法力系统

### 2.1 法力水晶

**基础规则：**
- 初始法力：0个可用/0个已填充
- 每回合开始：+1个最大法力（自动填充），直到上限10
- 法力上限：**10**（部分效果可提升至99）
- 每回合可用法力 = 已填充水晶数 - 被锁定数 + 临时法力

**法力水晶状态：**

| 状态 | 含义 |
|------|------|
| **空（Empty）** | 未填充的水晶槽位，本回合不可用 |
| **已填充（Filled）** | 已充能的水晶，本回合可用 |
| **被锁定（Overloaded）** | 因过载被锁定的水晶，不可用直到解锁 |

### 2.2 过载（Overload）机制

**触发流程：**
1. 打出带过载的卡牌时，卡牌文本中的过载值记录到 `overload_next`
2. 当前回合正常消耗法力
3. **下一回合开始时**：`overloaded = overload_next`，锁定对应数量的法力水晶
4. 再下一回合：锁定的水晶自动解锁

**关键规则：**
- 过载锁定的法力不可使用，即使你有足够的最大法力
- 过载不减少最大法力值，只锁定已填充的水晶
- 多次过载累加：`overload_next += 新值`
- 若锁定数 > 可填充数，超出部分顺延到后续回合

### 2.3 临时法力

**硬币/激活等效果：**
- 增加**临时法力水晶**（本回合可用，回合结束后消失）
- 临时法力可使当前可用法力**超过最大法力值**
- 临时法力不增加最大法力值
- 未使用的临时法力在回合结束时消失

### 2.4 法力消耗修改

**消耗增减规则：**
- "消耗减少N"效果叠加，最低降至0
- "消耗增加N"效果叠加
- 消耗修改在卡牌处于手牌时持续生效
- 卡牌离开手牌（打出/弃掉）后修改效果消失
- 消耗修改不改变卡牌的原始费用（original_cost不变）

**引擎实现要点：**
- `card.cost` 是动态值，受手牌buff影响
- `card.original_cost` 是静态值，不变
- 每次枚举合法动作时重新读取 `card.cost`

## 3. 战斗系统

### 3.1 攻击流程（6阶段详解）

每次攻击是一个**序列（Sequence）**，包含以下阶段：

| 阶段 | 名称 | 说明 |
|------|------|------|
| 1 | **提议攻击（Proposed Attack）** | 触发重定向/取消效果（冰冻陷阱、误导弹） |
| 2 | **攻击前触发（Pre-Attack）** | 非场面改变类触发（真银圣剑回血、祝福智慧抽牌） |
| 3 | **死亡结算** | 攻击前触发造成的死亡在此处理；攻击者/防御者死亡则取消攻击 |
| 4 | **伤害交换（Damage Exchange）** | **同时**造成伤害；攻击方触发器先结算，防御方后结算；武器-1耐久 |
| 5 | **攻击后触发（Post-Attack）** | 熊陷阱、收割机等效果 |
| 6 | **最终死亡结算** | 战斗死亡在此处理 |

**关键细节：**
- 攻击者和防御者**同时**造成伤害（各等于自身当前攻击力）
- 若攻击者在伤害交换前攻击力降为0，攻击仍发生但造成0伤害
- 若防御者在阶段3被移除（非死亡），攻击者**不会消耗攻击机会**，可重新攻击
- 英雄需要武器或临时攻击力 ≥ 1 才能攻击

### 3.2 嘲讽规则

**基础：** 对方场上有嘲讽随从时，必须先攻击嘲讽随从。
- 多个嘲讽随从时，玩家可选择攻击哪一个
- **嘲讽+潜行：** 潜行期间嘲讽被**压制**，可攻击其他目标
- **嘲讽+免疫：** 免疫期间嘲讽同样被**压制**
- 所有敌方嘲讽都被压制时，嘲讽规则完全不生效
- "无法攻击英雄"限制的是攻击者选目标，与嘲讽无关

### 3.3 冲锋（Charge） vs 突袭（Rush）

| 特性 | 冲锋 | 突袭 |
|------|------|------|
| 召唤当回合攻击随从 | ✅ | ✅ |
| 召唤当回合攻击英雄 | ✅ | ❌ |
| 后续回合 | 正常攻击 | 正常攻击 |
| 被沉默 | 恢复召唤失调 | 恢复召唤失调 |
| 获得方式 | 可后获得 | 可后获得 |

**冲锋细节：**
- 冲锋压制召唤失调但不移除它；沉默会恢复失调
- 冲锋来源在攻击准备阶段死亡不会取消攻击

**突袭细节：**
- 突袭仅在**召唤的那一回合**限制打脸
- 如果突袭随从在本回合不是新召唤的（如上回合就在场），可正常打脸

### 3.4 风怒（Windfury） / 超级风怒（Mega-Windfury）

**风怒：** 每回合最多攻击 **2次**（而非正常的1次）
- 第一次攻击后保留攻击能力，第二次攻击后消耗
- 攻击后获得风怒：可进行第二次攻击
- 攻击后失去风怒：不可进行第二次攻击
- 已攻击2次后重新获得风怒：仍不可攻击（已用尽配额）

**超级风怒：** 每回合最多攻击 **4次**
- 与风怒不叠加，取最大值

**与冰冻交互：**
- 风怒随从第一次攻击后被冰冻：失去后续攻击机会
- 冰冻解除规则：若角色还有未使用的攻击机会，回合结束解除；否则下回合结束解除

### 3.5 圣盾（Divine Shield）

**核心：** 吸收**第一次受到的任何伤害**，伤害归零后移除圣盾。

**不保护的情况：**
- 消灭效果（刺杀、扭曲虚空）
- 生命值修改（生平、猎人印记）
- 沉默、变形
- 非伤害类效果（冰冻仍会施加）

**关键交互：**
- **圣盾 vs 剧毒：** 圣盾吸收伤害 → 剧毒不触发（未实际受伤）→ 随从存活
- **0伤害事件：** 攻击0攻随从不移除圣盾
- **伤害+额外效果（如冰枪术）：** 伤害被吸收，但冰冻**仍施加**
- **不叠加：** 对已有圣盾的角色再次施加圣盾无效
- **伤害触发效果（如水元素冰冻）：** 伤害被吸收时不触发

### 3.6 剧毒（Poisonous）

**核心：** 被此来源伤害的**随从**被消灭（英雄不受影响）。
- 剧毒是伤害后的触发消灭效果
- **需要实际造成伤害**：圣盾格挡时剧毒不触发
- 0攻击力的剧毒随从不造成伤害，剧毒不触发
- 剧毒适用于该卡牌的**所有伤害来源**（战吼、亡语、触发效果）
- 剧毒武器（附魔致命毒药）：亡语伤害波也可触发剧毒

### 3.7 潜行（Stealth）

**何时打破潜行：**
- ✅ **攻击**时打破（伤害交换前）
- ✅ 对己方潜行随从施放法术/战吼不打破（己方可指定）
- ❌ 卡牌能力造成伤害（飞刀杂耍者）**不打破**潜行
- ❌ 受到伤害**不打破**潜行

**什么能绕过潜行：**
- ✅ AoE效果（暴风雪、烈焰风暴）
- ✅ 随机目标效果（拉格纳罗斯、飞刀杂耍者）
- ✅ 强制攻击效果（被迷惑的随从）
- ❌ 直接攻击（不可指定潜行随从）
- ❌ 定向法术/战吼/英雄技能（不可指定潜行随从）

**与嘲讽交互：** 潜行+嘲讽时，嘲讽被压制，潜行优先。

### 3.8 冰冻（Freeze）

**核心：** 被冰冻的角色失去**下一次攻击机会**。

**解除时机（关键规则）：**

| 场景 | 解除时机 |
|------|----------|
| 你的回合冰冻一个**还可以攻击但没攻击**的角色 | **你回合结束**时解除 |
| 你的回合冰冻一个**已经攻击过**的角色 | **你下回合结束**时解除 |
| 召唤失调/突袭无可攻击目标的新随从被冰冻 | **你下回合结束**时解除 |
| 风怒角色用了1次攻击后被冰冻 | **你回合结束**时解除（还有配额） |
| 风怒角色用了全部攻击后被冰冻 | **你下回合结束**时解除 |

**特殊规则：**
- 冰冻不阻止**被攻击**（防御时仍受伤害）
- 冰冻不阻止使用英雄技能、装备武器、打出卡牌
- 强制攻击效果（如伊利达雷审判官）**无视冰冻**
- 沉默可移除冰冻状态
- 打出英雄牌可解除英雄的冰冻

### 3.9 免疫（Immune）

**免疫保护：**
- ✅ 所有来源的伤害（战斗、法术、英雄技能、亡语、疲劳、友伤）
- ✅ 被敌方指定为目标（不可被敌方攻击/法术/战吼指定）
- ✅ 伤害触发效果不触发（因为伤害被阻止）

**免疫不保护：**
- ❌ 消灭效果（刺杀）
- ❌ 生命值修改（生平）
- ❌ 四骑士英雄技能秒杀（可杀死免疫英雄）
- ❌ 随机/自动目标效果仍可命中免疫角色（但伤害为0）

**与嘲讽交互：** 免疫+嘲讽时，嘲讽被压制。
**关键：** 沉默可移除免疫。

### 3.10 吸血（Lifesteal）

**核心：** 吸血来源造成的伤害等量恢复己方英雄生命值。
- 适用于**所有伤害来源**：战斗、战吼、亡语、触发效果、法术、英雄技能
- 先造成伤害，再恢复生命值
- **圣盾+吸血：** 造成0伤害 → 恢复0生命（但仍打破圣盾）
- **过量治疗：** 生命值不超过最大值上限
- **奥金尼灵魂祭司/暗影之拥：** 将吸血治疗转为伤害（不二次触发吸血，防止无限循环）

## 4. 关键词机制

### 4.1 战吼（Battlecry）

**触发条件：** 从手牌打出随从时触发。**其他方式召唤（复活、招募、随机效果）不触发。**

**目标选择：**
- 有目标的战吼：玩家**必须**选择一个合法目标，不可跳过
- 无合法目标时战吼部分不执行
- 选择目标时可右键取消整次打出

**执行顺序：**
1. 卡牌从手牌打出，法力消耗
2. **光环更新（生命/攻击）** — 新随从接受光环buff
3. **"每当你打出随从"触发器** 触发（如狙击秘密）
4. **战吼阶段** — 战吼效果结算（深度优先）
5. **"在你召唤之后"触发器** 触发（如飞刀杂耍者、镜像实体）
6. **光环更新 → 死亡创建 → 光环更新（其他）**

**铜须（Brann Bronzebeard）交互：**
- 铜须的光环使战吼触发**两次**
- 光环在战吼开始前检查；若铜须在战吼过程中被召唤，不追溯加倍
- 战吼中途目标被移除（返回手牌）：该部分失效

### 4.2 亡语（Deathrattle）

**触发条件：** 随从/武器/地标在**战场上被消灭**时触发。
- ❌ 弃掉（从手牌丢弃）不触发
- ❌ 烧毁（手牌满时抽牌）不触发
- ❌ 变形不触发（变形不经过死亡）

**死亡阶段（Death Phase）顺序：**
1. 所有生命值 ≤ 0 的随从**同时标记**为待死亡
2. 死亡事件按**出场顺序**排队（先出场的先结算）
3. 每个死亡事件内：亡语 + 死亡触发秘密 + 死亡触发效果 → 全部按出场顺序排列
4. 救赎（Redemption）有**特殊优先级**：始终排在该死亡事件的最后

**亡语队列规则：**
- 队列**不可变**：结算开始后不添加新触发器
- **深度优先**结算：一个亡语引发新事件时，先完成新事件再回到原队列
- 级联亡语（亡语杀死另一个亡语随从）：新死亡记入但不立即处理，当前阶段结束后再处理

**巴隆·里维endas（Baron Rivendare）：**
- 光环使亡语触发**两次**
- 光环在亡语事件开始前检查，不可追溯

**沉默：** 沉默移除所有亡语附魔，被沉默的随从死亡时不触发亡语。
**复生（Reborn）：** 亡语先触发，然后随从以1点生命值复活。

### 4.3 发现（Discover）

**核心规则：** 从3个随机选项中选择1张加入手牌。
- 选项池：默认为**中立 + 当前职业卡牌**
- 无重复选项（3张互不相同）
- 必须选择一张（超时随机选）
- 不可发现自身的副本

**池过滤规则：**
- "发现一张法术" → 仅法术牌
- "发现一张(D种族)随从" → 仅该种族随从
- "发现另一职业的卡牌" → 仅其他职业卡牌（不含中立）
- "发现任意卡牌" → 所有职业 + 中立

**发现时机：** 选中的卡牌在**当前序列结束时**加入手牌。
- 手牌满时：卡牌等待，有空间时加入

**与黑暗之赐交互（Dark Gift）：**
- 死骑、恶魔猎手、潜行者、术士、战士发现随从时
- 每个选项额外获得一个随机黑暗之赐buff
- 玩家选择的是"卡牌+buff"的组合

### 4.4 抉择（Choose One）

**核心：** 德鲁伊专属机制，打出时选择两种互斥效果之一。
- 抉择**不是战吼**：不受铜须加倍、不受蛛魔网行者加费
- 变形类抉择（如利爪德鲁伊）：回手后保持变形状态
- 非变形类抉择：回手后重置

**范达尔·鹿盔（Fandral Staghelm）：**
- 光环效果：抉择卡牌**同时获得两种效果**

### 4.5 连击（Combo）

**核心：** 如果本回合已打出过其他卡牌，该卡获得额外效果。
- 第一张打出的牌不触发连击
- 之后的每张牌都触发连击（不限次数）
- 追踪的是"本回合打出的卡牌数"，不是"连击"关键词本身

### 4.6 激励（Inspire）

**触发条件：** 英雄技能使用后触发，按出场顺序结算。
- 英雄技能效果**先完成**，然后激励触发
- 激励阶段新加入的激励随从**不触发**（队列不可变）
- 英雄技能杀死英雄时：激励不触发（游戏已结束）

### 4.7 过载（Overload）

详见 [2.2 过载机制](#22-过载overload机制)。

### 4.8 流放（Outcast）

**核心：** 恶魔猎手专属。卡牌在手牌**最左或最右**时打出获得额外效果。
- 检查时机：打出时检查手牌位置
- 不在最左/最右时：仅基础效果
- 鼓励玩家策略性管理手牌位置

### 4.9 复生（Reborn）

**核心：** 随从死亡后以**1点生命值**复活一次。
- 复活保留所有附魔（但生命值重置为1）
- 亡语在复生前触发（先死亡 → 亡语 → 复生）
- 复生后的随从是同一个游戏对象（不是新召唤）
- 沉默移除复生关键词

### 4.10 超杀（Overkill）

**核心：** 对随从造成**超过其剩余生命值**的伤害时触发。
- 只在己方回合触发
- 伤害溢出量不影响触发（溢出1点和溢出10点效果相同）
- 多个超杀效果分别触发

### 4.11 腐蚀（Corrupt）

**核心：** 手牌中的腐蚀牌在你打出一张**更高费用**的牌时升级。
- 比较的是当前费用（含修改），非原始费用
- 升级后保留手牌buff（如减费效果保持）
- 升级后的卡牌视为"不是从套牌开始的牌"

## 5. 触发与光环系统

### 5.1 阶段（Phase）与序列（Sequence）

**核心概念：** 炉石传说的所有游戏行为都通过"序列→阶段"结构执行。

- **序列（Sequence）：** 一个玩家动作（攻击、打出卡牌、英雄技能等）产生一个序列
- **阶段（Phase）：** 序列内的子步骤，深度优先执行
- **最外层阶段结束时**才执行三步清理：
  1. 光环更新（生命/攻击）
  2. 死亡创建步骤
  3. 光环更新（其他）

**关键规则：**
- 死亡不在阶段中间处理，只在最外层阶段结束
- 子阶段内的死亡累积到最外层阶段结束统一处理
- 阶段开始时快照触发器队列，之后不可变

### 5.2 "每当你"（Whenever） vs "在你之后"（After）触发

| 方面 | "每当你"触发 | "在你之后"触发 |
|------|-------------|---------------|
| 法术时机 | 法术**开始施放时**触发 | 法术**完全结算后**触发 |
| 随从时机 | 随从**打出时**触发（战吼前） | 随从**召唤完成后**触发（战吼后） |
| 示例 | 法术反制、大法师安东达斯 | 狂野火法师、任务达人 |
| 能否阻止事件 | ✅ 可阻止（法术反制） | ❌ 不能（事件已发生） |

**触发器队列规则：**
- 同一事件的所有触发器按**出场顺序**排队
- 队列在开始结算时**冻结**，不添加新触发器
- 深度优先：一个触发器引发新事件时，先完成新事件再回原队列

### 5.3 光环效果（Aura）

**核心：** 光环是持续性效果，在光环源在场期间为目标提供**临时附魔**。

**特性：**
- **动态重算：** 每次光环更新步骤重新计算，新进入场的随从自动获得buff
- **应用顺序：** 永久buff先计算，光环buff叠加在最上层
- **自身受益：** 若光环源满足自身条件，也会获得buff

**光环源离场：**
- 光环附魔在**下一次光环更新步骤**时移除
- 不在随从离场的瞬间移除（延迟到清理步骤）
- 沉默光环源：移除光环效果
- 沉默被buff的随从：**不**移除光环附魔（光环会重新施加）

**光环更新步骤（Aura Update）：**
1. **光环更新（生命/攻击）：** 处理修改属性的光环，附魔列表重排（光环附魔移到最后）
2. **死亡创建步骤：** 检查并处理生命值 ≤ 0 的随从
3. **光环更新（其他）：** 处理非属性光环（铜须、里维endas等）

**光环源被窃取（精神控制）：**
- 不立即为新方释放光环，等最外层阶段结束
- 除非新方有新随从入场（触发重算）

### 5.4 回合开始/结束触发

**回合开始（Start of Turn）：**
1. 回合开始触发器按**出场顺序**结算
2. **抽牌步骤**：从牌库顶部抽1张牌
3. **疲劳检查**：牌库为空时造成疲劳伤害（见第8章）
- 回合开始触发在抽牌**之前**结算
- 回合开始效果杀死的随从在阶段结束后处理死亡

**回合结束（End of Turn）：**
1. 回合结束触发器按**出场顺序**结算
2. 队列在阶段开始时固定，新加入的回合结束效果**不加入本回合队列**
- 克尔苏加德复活的随从若有回合结束效果：本回合不触发（队列已固定）
- 死亡在子阶段间和最外层阶段结束时处理

### 5.5 附魔系统（Enchantment）

**附魔属性：**
- 来源卡牌ID、名称
- 属性修改：+攻击力、+生命值、+最大生命值、+费用
- 关键词增删：添加/移除关键词
- 触发类型：亡语、回合结束、回合开始、攻击时、光环
- 持续时间：-1（永久）、>0（回合数）

**附魔规则：**
- 跨区域移动时附魔脱落
- 手牌附魔（如减费）在打出后保留在生成的随从上
- 沉默移除**所有**附魔
- 变形移除**所有**附魔（替换为新游戏对象）
- 亡语附魔可与卡牌自带亡语共存（多个亡语叠加）

## 6. 奥秘系统

### 6.1 奥秘基础规则

**核心：** 以法术形式打出，效果隐藏，满足条件时在对方回合自动触发。

| 规则 | 说明 |
|------|------|
| **上限** | 最多5个不同奥秘同时激活 |
| **唯一性** | 同名奥秘不可重复打出 |
| **触发时机** | 默认对方回合，例外：竞争精神/打开兽栏在我方回合开始时 |
| **触发顺序** | 多个奥秘同时触发时按**打出顺序**结算 |
| **法术伤害** | 法术伤害修正适用于造成伤害的奥秘（如爆炸陷阱） |

**特殊规则：**
- 奥秘在对方**战吼结算后**触发（随从打出类奥秘）
- 一个奥秘的结算移除后续奥秘的目标时，后续奥秘**失效**
- 目标已致命/待死亡的奥秘**不触发**（Rule 5）
- 奥秘不是随从，不占战场位置，不触发随从相关效果

### 6.2 各职业奥秘触发条件

**法师（Mage）奥秘：**
| 触发类型 | 奥秘示例 | 触发条件 |
|----------|---------|----------|
| 攻击触发 | 冰冻陷阱 | 敌方随从攻击时 |
| 法术触发 | 法术反制、法术转移 | 对方施放法术时 |
| 随从触发 | 狙击、镜像实体 | 对方打出随从时 |
| 伤害触发 | 躲避 | 英雄受到伤害时 |

**猎人（Hunter）奥秘：**
| 触发类型 | 奥秘示例 | 触发条件 |
|----------|---------|----------|
| 攻击触发 | 冰冻陷阱、毒蛇陷阱 | 敌方随从攻击时 |
| 随从触发 | 狙击 | 对方打出随从时 |
| 伤害触发 | 爆炸陷阱 | 对方造成伤害时 |

**潜行者（Rogue）奥秘：**
- 随从触发、攻击触发等（随版本变化）

**法术反制（Counterspell）特殊规则：**
- 作为"每当"触发器，在法术**开始施放时**拦截
- 法力已消耗，但法术效果**完全阻止**
- 阻止后"在你施放法术后"触发器**不触发**
- 在法术反制之前打出的其他奥秘（如法力绑定）仍可触发

## 8. 卡牌抽取与疲劳

### 8.1 抽牌规则

**每回合抽牌：**
- 回合开始触发器结算后，从牌库顶部抽1张牌
- 先手玩家第1回合不抽牌，后手玩家第1回合抽1张

**抽牌触发器：**
- "每当你抽牌"触发器在卡牌加入手牌时触发
- 抽到的牌可见后才触发相关效果

**抽多张牌：**
- 逐张抽取，每张牌独立触发抽牌触发器
- 手牌满时：超出部分直接进入墓地（见8.2）

### 8.2 过载抽牌（Overdraw）

**手牌上限：** 10张

**手牌满时抽牌：**
- 第11张及以后的牌**直接进入墓地**（不是弃牌，不触发弃牌效果）
- 这些牌的信息会显示给玩家（"烧毁"提示）
- 烧毁的牌**不触发亡语**（亡语仅在战场死亡时触发）
- 烧毁的牌**不是死亡事件**，不进入死亡事件缓存

**引擎实现要点：**
- 抽牌时先检查 `len(hand) >= 10`
- 若满：牌直接进墓地，返回烧毁标记
- 若未满：`hand.append(card)`

### 8.3 疲劳伤害（Fatigue）

**触发条件：** 抽牌时牌库为空。

**伤害规则：**
- 首次疲劳：**1点**伤害
- 之后每次疲劳伤害 **+1**（2、3、4、...）
- 疲劳伤害**不可回避**（免疫不保护，护甲可吸收）
- 疲劳伤害直接对英雄造成（不经过随从）

**引擎实现：**
- `fatigue_damage` 计数器从0开始
- 每次触发疲劳：`fatigue_damage += 1`，然后 `hero.hp -= fatigue_damage`
- 疲劳计数器不重置（整场游戏持续累加）

## 9. 2026标准模式特殊机制

### 9.1 兆示（Herald）

**来源：** 大地的裂变（CATACLYSM，2026年3月）
**适用职业：** 死亡骑士、恶魔猎手、潜行者、萨满、术士、战士（死亡之翼阵营）

**核心规则：**
- 打出带有兆示关键词的牌时：**召唤1个士兵**（Soldier），并递增兆示计数器
- 士兵的属性/能力与职业巨型(Colossal)随从的附肢相同
- **升级阈值：**
  - 累计2次兆示 → 巨型随从和附肢属性**翻倍**
  - 累计4次兆示 → 再次翻倍（4倍基础）
- 死亡之翼英雄牌同步升级：2次兆示解锁2个灾变，4次解锁全部4个灾变

**引擎需要追踪：**
- `herald_count: int` — 全局计数器
- `colossal_upgrade_level: int` — 0/1/2（对应0/2/4次兆示）
- 士兵模板表（按职业）

### 9.2 裂变（Shatter）

**来源：** 大地的裂变（CATACLYSM，2026年3月）
**适用职业：** 德鲁伊、猎人、法师、圣骑士、牧师（守护巨龙阵营）

**核心规则：**
- 裂变牌**仅限法术**，进入手牌时**分裂为两张半牌**
- 左半牌放置在手牌**最左端**，右半牌放置在**最右端**
- 每张半牌：**全额费用**但**半数效果**
- 两张半牌在手牌中**相邻时**合并为组合牌
- 组合牌：全额费用，**合并两半效果**（通常非常强力）

**关键规则：**
- 手牌满时：裂变牌**不分裂**（无空间放两张）
- 手牌只有1个空位：只放入左半牌
- 合并后**移除裂变关键词**（洗回/复制不再分裂）
- 合并时两半的手牌附魔叠加（如减费效果合并为双重减费）

**引擎需要追踪：**
- 手牌中每张牌的**索引位置**（不再是无序列表）
- 裂变半牌的配对关系
- 相邻检测：每次手牌变化时检查

### 9.3 延系（Kindred）

**来源：** 安戈洛龟途（Lost City of Un'Goro，2025年7月）

**核心规则：**
- 带有延系关键词的牌：如果你**上一回合**打出了**相同种族或法术派系**的牌，获得额外效果
- 延系在从手牌打出时检查，不是召唤时
- **双种族随从**只需匹配其中一个种族
- 延系不是战吼（除非文本写"战吼：延系改为..."，此时铜须可加倍）

**引擎需要追踪：**
- `last_turn_played_races: Set[str]` — 上一回合打出的种族集合
- `last_turn_played_schools: Set[str]` — 上一回合打出的法术派系集合
- 每回合结束时快照 `cards_played_this_turn` 的种族/派系

### 9.4 回溯（Rewind）

**来源：** 时光英雄（Across the Timeways，2025年11月）

**核心规则：**
- 打出带有随机效果的法术后，提示"保留"或"回溯"
- 选择**回溯**：游戏状态回退到打出该牌之前，然后立即重新打出（新随机结果）
- 回溯过程中**不能打出其他牌**
- 可能得到相同结果（随机种子可能重复）
- 多张回溯牌连续打出时，顺序执行

**特殊卡牌 — Morchie（Echoes迷你系列）：**
- 回溯时**保留两个结果**（不回退，两个效果都生效）

**关键限制：**
- 若回溯牌导致英雄死亡：不提供回溯选项
- 回合计时器在回溯动画期间继续计时

**引擎实现：**
- 需要状态快照/恢复能力
- 在RHEA评估中：对回溯牌执行2次分支评估，取较好结果

### 9.5 黑暗之赐（Dark Gift）

**来源：** 漫游翡翠梦境（Into the Emerald Dream，2025年3月）
**适用职业：** 死亡骑士、恶魔猎手、潜行者、术士、战士

**核心规则：**
- 这些职业在发现(Discover)随从时，每个选项额外获得一个随机黑暗之赐buff
- 始终展示**3个不同的**黑暗之赐
- 黑暗之赐在发现时应用，不是打出时

**10种黑暗之赐效果：**

| 名称 | 效果 |
|------|------|
| 梦魇苏醒 | +4/+5，放在牌库顶部 |
| 羽毛利爪 | 圣盾 + 风怒 |
| 短爪 | -2费 -2攻击力 |
| 粗壮包裹 | +4生命值 + 嘲讽 |
| 美梦正酣 | +2/+2 + 魔免 |
| 清醒梦魇 | 召唤一个2/2复制 |
| 睡眠行者 | 冲锋（仅攻击力≥1时） |
| 不灭恐惧 | 复生（满血复活） |
| 粗鲁唤醒 | 战吼触发两次（仅原有战吼时） |
| 梦游惊魂 | +3攻击力 + 吸血 |

### 9.6 传说英雄（Fabled）

**来源：** 时光英雄（Across the Timeways，2025年11月）

**核心规则：**
- 传说英雄牌是**卡牌包**：打出时附带2+张不可收集的传说衍生物
- 衍生物**计入30张套牌上限**（如包含3张衍生物，剩余只有27个槽位）
- 全有或全无：不可选择性排除衍生物
- 传说英雄牌**不可被随机发现或生成**
- 传说英雄牌之间有协同效果（打出其他传说英雄牌增强效果）

**引擎实现：**
- 套牌构建时检测Fabled标签，自动添加衍生物
- 追踪已打出的传说英雄牌（用于协同效果）

### 9.7 巨型（Colossal）v2

**来源：** 首次出现在沉没之城，大地的裂变回归并配合兆示升级

**核心规则：**
- 格式："巨型+N"（N为附臂数量）
- 随从**召唤或变形**时，同时召唤N个附肢到主体右侧
- 附肢是独立随从，有独立属性和效果
- 附肢与主体**共享7个战场位置**
- 空间不足时：超出部分的附肢**不召唤**
- 主体被消灭不影响已召唤的附肢

**大地的裂变新增：**
- 每个职业1张巨型传说随从
- 6张配合兆示升级（兆示2次/4次翻倍附肢属性）
- 5张为标准巨型（无兆示联动）

**引擎实现：**
- 解析 "巨型+N" 获取附臂数量
- 召唤时按位置插入：主体 + N个附肢
- 兆示升级时附肢属性乘以升级倍数

### 9.8 休眠（Dormant）

**核心规则：**
- 休眠随从作为**永久物**打出（不可攻击，免疫大部分效果）
- 回合倒计时：每个己方回合开始时计数器-1
- 计数器归零时**觉醒**，获得卡牌属性和所有已积累的附魔
- 休眠期间保留被施加的附魔
- 复制已觉醒随从 → 产出已觉醒的副本

**关键限制：**
- 无自我觉醒机制的休眠随从**不可被随机召唤**（防止永久无用召唤）
- 沉默效果交互但通常不唤醒
- 变形效果完全替换休眠随从

### 9.9 手牌指定（Hand Targeting）

**来源：** 大地的裂变（CATACLYSM，2026年3月）— 全新技术

**核心规则：**
- 部分卡牌可以指定**手牌中的任意一张牌**作为目标
- 之前的类似效果只能从3个随机选项中选择
- 大地的裂变引入了**直接指定**版本，未来将扩展使用

**示例：** 龙穴大厅（Chamber of Aspects）地标 — 指定一个手牌随从+2/+2

**引擎实现：**
- 新增动作类型：`TARGET_HAND_CARD`
- 需要 `source_card_index` 和 `target_card_index`
- 枚举合法动作时生成所有合法的手牌目标组合

### 9.10 任务（Quest）系统

**核心规则：**
- 1费法术，**始终出现在起手手牌**
- 打出后追踪进度（如"打出N张法术"、"召唤N个野兽"）
- 进度在**双方回合**都计算
- 达成时获得奖励卡牌（在当前序列结束后加入手牌）
- 手牌满时奖励等待空位

**关键限制：**
- 每位英雄同时只能有**1个**活跃任务
- 特殊卡牌可绕过此限制
- 任务不可被随机发现/生成（但可被复制）
- 任务完成时机：**当前阶段结束后**生成奖励（不会被同阶段的弃牌效果消耗）

**引擎需要追踪：**
- `active_quests: List[QuestState]`
- 进度计数器 + 阈值
- 任务类型（法术、随从、攻击等触发条件）

## 10. 附录：阶段解析与规则引擎

### 10.1 完整阶段解析流程

**打出随从的完整流程：**
```
1. 从手牌移除卡牌，消耗法力
2. 光环更新（生命/攻击）→ 新随从接受光环
3. "每当你打出随从"触发器（秘密：狙击、镜像实体）
4. 战吼阶段（深度优先结算）
   ├── 选择目标（若需要）
   ├── 铜须检查（光环在开始前确认）
   └── 战吼效果执行
5. "在你召唤之后"触发器（飞刀杂耍者）
6. [最外层阶段结束]
   ├── 光环更新（生命/攻击）
   ├── 死亡创建步骤
   │   ├── 标记所有生命≤0的随从
   │   ├── 按出场顺序排亡语队列
   │   └── 逐个执行死亡事件（深度优先）
   └── 光环更新（其他）
```

**攻击的完整流程：** 见 [3.1 攻击流程](#31-攻击流程6阶段详解)

### 10.2 死亡创建步骤（Death Creation Step）

**触发时机：** 仅在最外层阶段结束时执行。

**流程：**
1. 扫描所有角色，标记 `health ≤ 0` 或 `pending_destroy` 的
2. 按**出场顺序**排队
3. 每个死亡事件创建独立的子阶段：
   - 收集该随从的所有亡语 + 死亡触发秘密 + 死亡触发效果
   - 按出场顺序排列（救赎排最后）
   - 深度优先执行
4. 子阶段结束后再次检查新死亡（级联）
5. 最多级联5层（防止无限循环）

### 10.3 光环更新步骤

**三步清理（在最外层阶段结束时）：**

1. **光环更新（生命/攻击）：**
   - 移除所有属性类光环附魔
   - 重新应用所有活跃光环
   - 重算所有随从的最终属性

2. **死亡创建步骤：** 见10.2

3. **光环更新（其他）：**
   - 处理非属性光环（铜须、里维endas、法术伤害等）
   - 这些光环影响下一个事件的处理方式

### 10.4 触发队列不可变性

**核心规则：** 触发器队列在开始结算时冻结。

**含义：**
- 新加入的触发器**不加入**当前队列
- 深度优先：一个触发器引发新事件时，新事件有自己独立的队列
- 返回原队列时，继续按原顺序执行

**防止无限循环的机制：**
- 队列不可变性自然限制级联
- 死亡延迟处理（不在阶段中间）打破循环链
- 大部分触发器"每个事件仅触发一次"
- 游戏引擎有最终安全阀（极端情况强制终止）

### 10.5 对决策引擎的实现意义

**引擎架构关键映射：**

| 游戏规则 | 引擎实现 |
|----------|----------|
| 最外层阶段结束才处理死亡 | `apply_action` 后统一 `_resolve_deaths()` |
| 触发队列按出场顺序 | `board` 列表的索引天然保证顺序 |
| 光环动态重算 | 每次状态变化后重算所有光环附魔 |
| 深度优先触发 | 递归处理，限制最大深度5层 |
| 手牌位置重要 | 手牌用有序列表+索引，不用集合 |
| 全局状态计数器 | `GameState` 新增 `herald_count`、`imbue_level` 等 |
| 回合历史追踪 | `last_turn_played_races`、`last_turn_played_schools` |

**优先级总结（P0 = 必须实现）：**

| 优先级 | 机制 | 影响卡牌数 | 实现难度 |
|--------|------|-----------|----------|
| **P0** | 战吼解析+效果 | 334 | 中 |
| **P0** | 亡语队列 | 139 | 中 |
| **P0** | 附魔/光环系统 | 全局 | 高 |
| **P0** | 发现框架 | 80 | 中 |
| **P1** | 奥秘触发 | 少量 | 低 |
| **P1** | 英雄技能(Imbue) | 19 | 中 |
| **P1** | 地标牌 | 17 | 低 |
| **P1** | 手牌位置系统 | 多机制 | 中 |
| **P2** | 兆示/巨型 | 14+14 | 中 |
| **P2** | 裂变 | 少量 | 高 |
| **P2** | 回溯 | 19 | 高 |
| **P2** | 延系 | 29 | 低 |
| **P2** | 任务追踪 | 14 | 低 |
| **P2** | 黑暗之赐 | 20 | 低 |
